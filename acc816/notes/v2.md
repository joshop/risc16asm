This is a simple processor with:

- 8-bit datapath
- 16-bit addresses
- Instruction register
- Simple ALU

Parts:

- IC EEPROM 256KBIT PARALLEL 28BIT [2x]
- IC SRAM 256KBIT PARALLEL 28DIP [2x]

Registers

- x0: always zero
- x1: 8-bit temporary register
- x2: address high
- x3: address low

Instructions

- clear carry
- load upper immediate
- addi (add signed immediate)
  - if C control bit on, use C and carry result gets stored in C flag
- add (rd <- rs1 + rs2)
  - if C control bit on, use C and carry result gets stored in C flag
- nandi
- nand
- load
  - load from memory address `{x2,x3}+imm` to any register
- store
  - store x1 into memory address `{x2,x3}+imm`
- branch by [signed imm] if carry set
  - shared 16-bit adder for PC
- branch by [signed imm] if zero
- jump to {x2,x3}

Pseudoinstructions

- li rd, imm
  - lui rd, imm_HI
  - addi rd, imm_LO
- load rd, mem[imm]
  - li {x2,x3}, imm
  - load rd, {x2,x3}

More complex stuff

- How do we do mem[C] <- mem[A] + mem[B]?
  - Suppose A, B, are literals
  - 16-bit aligned values
  - Stuff should already be pushed onto the stack, so we can use all registers freely
  - Procedure
    - {x2,x3} <- A (addi x2)
    - x1 <- mem[{x2,x3}] (load)
    - {x2,x3} <- B (addi x2)
    - x2 <- mem[{x2,x3}] (load)
    - x1 <- x1 + x2 (add)
    - {x2,x3} <- C (addi x2)
    - mem[{x2,x3}] <- x1 (store)
    - (do it all over again but with A+1, B+1, and with carry flag)
